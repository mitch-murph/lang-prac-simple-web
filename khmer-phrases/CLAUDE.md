# Khmer Audio Phrase Extractor тАФ Project Context

## What this project does

This project scrapes Khmer travel vocabulary from a saved HTML page (loecsen.com), downloads the Khmer audio files, generates a CSV of all phrases, and provides a local web UI for filtering, previewing, and building "drill" audio files (normal speed тЖТ pause тЖТ slow тЖТ pause тЖТ normal).

## Pages

### `admin.html` тАФ Settings / Config
- Loads `khmer_phrases.csv` automatically via `fetch`
- Shows all phrases grouped by category with show/hide toggles (replaces the old checkbox "selected" concept)
- Also has drill settings (slow speed %, pause duration) тАФ defaults 87% speed, 2s pause
- Exports `khmer_phrases_filtered.csv` with a `Visible` column for ALL rows (true/false)
- Saves the visible/hidden state to `sessionStorage` so viewer.html can immediately consume it without needing a file

### `viewer.html` тАФ Content viewer
- Tries `sessionStorage` first (set by admin.html) тАФ works while server is running
- Falls back to fetching `khmer_phrases_filtered.csv` from the same directory тАФ works on S3 static hosting
- Only shows phrases where `Visible = true`
- Has play (тЦ╢) and play-drill (ЁЯФБ) buttons per row
- No download/export functionality тАФ this is the end-user facing page

## Audio file naming

Files are saved as: `English phrase (phonetic).mp3`

- If the same English phrase appears multiple times in a category (e.g. "Yes" has male/female variants), the phonetic is appended for all: `Yes (chars).mp3`, `Yes (bat).mp3`
- If even the phonetics clash, a counter is appended: `Yes (chars).mp3`, `Yes (chars) 2.mp3`
- Directories are zero-padded by category index so they sort correctly: `01 Essentials`, `02 Conversation`, etc.

## Drill audio format

When playing or downloading drill audio, the format is:

```
1s silence тЖТ normal speed тЖТ [pause] тЖТ slow speed тЖТ [pause] тЖТ normal speed тЖТ 1s silence
```

- Slow speed and pause duration are configurable in the admin UI
- Defaults: 87% speed, 2s pause
- Slow version is generated via linear interpolation in the browser (no server needed)
- Drill MP3s are encoded client-side using [lamejs](https://github.com/zhuker/lamejs)

## Audio source

Original audio hosted at:
```
https://www.loecsen.com/OrizonFlash_V2/ressources/son/khmer-{n}-{m}.mp3
```
The `data-id` attributes in the source HTML are the filenames (e.g. `khmer-1-1.mp3`).

## S3 deployment notes

`viewer.html` is fully static тАФ it fetches `khmer_phrases_filtered.csv` relative to its own URL. To deploy:

1. Run `npm run extract` to generate audio + CSV
2. Open `admin.html` via `npm start`, configure which phrases are visible, export `khmer_phrases_filtered.csv`
3. Upload to S3:
   - `viewer.html`
   - `khmer_phrases_filtered.csv`
   - `audio/` directory

No backend required. The viewer page reads the CSV directly.
